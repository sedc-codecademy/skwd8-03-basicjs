Comparison operators

Comparison operators

  • The result is always a boolean value
Operator Name Example Explanation
> Greater than operator a > b Left operand is greater than the right operand
>= Greater than or equal operator a >= b Left operand is greater than or equal to the right operand
< Less than operator a < b Left operand is less than the right operand
<= Less than or equal operator a <= b Left operand is less than or equal to the right operand

Equality Operators

  • The result is always a boolean value
Operator Name Example Explanation
== Equality a == b Checks if the operands are not equal. Converts the operands if they are not of the same type, then applies strict comparison
!= Inequality a != b Checks if the operands are not equal. Converts the operands if they are not of the same type, then applies strict comparison
=== Identity / strict equality a === b The identity operator returns true if the operands are strictly equal (see above) with no type conversion
!== Non-identity / strict inequality a !== b The non-identity operator returns true if the operands are not equal and/or not of the same type

Logical Operators

  • Logical operators are typically used with Boolean (logical) values
    • When they are, they return a Boolean value
  • When used with non-Boolean values, they may return a non-Boolean value
Operator Name Example Explanation
&& Logical AND expr1 && expr2 Returns expr1 if it can be converted to false, otherwise, returns expr2. Thus, when used with Boolean values, && returns true if both operands are true; otherwise, returns false.
|| Logical OR expr1 || expr2 Returns expr1 if it can be converted to true, otherwise, returns expr2. Thus, when used with Boolean values, || returns true if either operand is true.
! Logical NOT !expr Returns false if its single operand can be converted to true; otherwise, returns true

Logical AND (&&)

                    


a1 = true  && true       // t && t returns true
a2 = true  && false      // t && f returns false
a3 = false && true       // f && t returns false
a4 = false && (3 == 4)   // f && f returns false
a5 = 'Cat' && 'Dog'      // t && t returns "Dog"
a6 = false && 'Cat'      // f && t returns false
a7 = 'Cat' && false      // t && f returns false
a8 = ''    && false      // f && f returns ""

Logical OR (||)

                    


o1 = true  || true       // t || t returns true

o7 = 'Cat' || false      // t || f returns "Cat"
o8 = ''    || false      // f || f returns false

Special Cases and equality

Special Numbers

  • Any mathematic operation you perform without both operands being numbers (or values that can be interpreted as regular numbers) will result in producing "not a number" or NaN
  • The type of not-a-number is 'number'! Hooray for confusing names and semantics!

Special Numbers


var a = 2 / "foo";        // NaN

typeof a === "number";    // true

//Beware
a == NaN;    // false
a === NaN;    // false
isNaN( a ); // true
//buuut
var b = "foo";
isNaN( b ); // true -- ouch!
//ES6 to the rescue
//finally
Number.isNaN(..) // typeof n === "number" && window.isNaN( n )

Infinities


var a = 1 / 0;    // Infinity 
var b = Infinity; // Infinity
var c = -1 / 0;    // -Infinity
var d = -Infinity; // -Infinity

== or === dilema

  • If either value (aka side) in a comparison could be the true or false value, avoid == and use ===
  • If either value in a comparison could be of these specific values (0, "", or [] -- empty array), avoid == and use ===
  • In all other cases, you're safe to use ==. Not only is it safe, but in many cases it simplifies your code in a way that improves readability.

Switch statements

  • Starts with a variable called switch value
  • Each case indicates a possible value for that variable
  • Have a default option if no match is found

Structure


switch(someVal) {
	case 'value1':
		//do something;
		break;
	case 'value2':
		//do something else;
		break;
	default:
		//no value match found, do default option
		break; //this is optional
}

"fall through" scenario

  • If you omit break from a case, and that case matches or runs, execution will continue with the next case's statements regardless of that case matching


switch(someVal) {
	case 'value1':
	case 'value2':
		//do something;
		break;
	case 'value3':
		//do something else;
		break;
	default:
		//no value match found, do default option
		break; 
}
